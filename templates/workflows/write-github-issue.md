# Workflow: Write GitHub Issue

## Goal

Turn a feature description or bug report into a structured GitHub issue that is actionable for developers.

## Process

1. **Receive Input:** The user provides a feature idea, bug report, or improvement request.
2. **Clarify (if needed):** Ask 1–3 clarifying questions when the scope or impact is unclear.
3. **Structure the Issue:** Format as a GitHub issue with the sections below.
4. **Output:** Provide the issue body as markdown or paste-ready text.

## Issue Structure

### For Features

```markdown
## Summary
[One-paragraph description of the feature]

## Problem
[What problem does this solve? Who is affected?]

## Proposed Solution
[Describe the proposed behavior and implementation approach]

## Acceptance Criteria
- [ ] [Criterion 1]
- [ ] [Criterion 2]
- [ ] [Criterion 3]

## Out of Scope
[What this feature will NOT include]

## Additional Context
[Links, mockups, related issues]
```

### For Bug Reports

```markdown
## Summary
[Brief description of the bug]

## Steps to Reproduce
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Expected Behavior
[What should happen]

## Actual Behavior
[What actually happens]

## Environment
- OS: [e.g. macOS 14]
- Version: [e.g. 1.2.3]
- [Other relevant info]

## Additional Context
[Logs, screenshots, related issues]
```

## Output

- Markdown-ready issue body.
- Clear title suggestion.
- Appropriate labels (if the user can suggest them).
